## 1. データウェアハウス（DWH）とは

意思決定のために、社内の膨大なデータを整理・蓄積しておくための**「巨大なデータの倉庫」**です。

- **主な役割:** 異なるシステムからデータを集め（統合）、過去の履歴を消さずに残し（蓄積）、分析しやすい形に整えること。  
- **特徴:** 時系列でデータを保持し、一度格納したデータは基本的に更新（上書き）しません。

### 似た言葉との違い（データレイク・データマート）

よく比較される用語を整理しました。

| **用語**     | **状態**   | **特徴**                                  |
| ---------- | -------- | --------------------------------------- |
| **データレイク** | 生の状態     | 加工前のデータをそのまま放り込む「池」。自由度は高いが整理されていない。    |
| **DWH**    | 整えられた状態  | 目的のために整理・構造化された「倉庫」。分析のメイン拠点。           |
| **データマート** | 切り出された状態 | DWHから「営業部用」「マーケ用」など、特定の目的のために小分けした「売店」。 |

## 2. データベース（DB）との違い

「今を動かすためのDB」と「過去を振り返るためのDWH」という役割の違いがあります。

| **項目**     | **データベース (DB)**  | **データウェアハウス (DWH)** |
| ---------- | ---------------- | ------------------- |
| **目的**     | 日々の業務処理（受注・在庫更新） | 分析・意思決定（売上予測・傾向把握）  |
| **データの鮮度** | 「今」の最新状態が重要      | 「過去から現在」の蓄積が重要      |
| **主な処理**   | 頻繁な書き換え・削除       | データの追加と大量集計         |
| **負荷の影響**  | レジやアプリの動作に直結     | 分析レポートの表示速度に直結      |

---

## 3. データマートとデータベース（DB）の違い

**データマートとデータベースは「技術的な中身」は似ていますが、「役割」が違います。**
もっと言うと、**「データマートは、データウェアハウス（DWH）から特定の目的のために切り出された、ミニデータベース」** だと考えると分かりやすいです。

| **項目**       | **データベース (DB)** | **データマート (Mart)**       |
| ------------ | --------------- | ----------------------- |
| **役割**       | システムを動かすための「土台」 | 特定の部署が分析するための「道具」       |
| **データの範囲**   | システムに必要な全データ    | **「営業部だけ」「先月分だけ」**など限定的 |
| **作成のタイミング** | アプリやサービスを作る時    | DWHが巨大になりすぎて使いにくい時      |
| **利用イメージ**   | レジやWebサイトの裏側    | ExcelやBIツール（グラフ作成）の裏側   |
### なぜデータマートが必要なの？（DWHとの関係）

DWHは「社内の全データが揃っている巨大な倉庫」です。便利ですが、いくつか欠点があります。

1. **デカすぎて探すのが大変：** 迷子になります。
2. **計算に時間がかかる：** 全データから探すと時間がかかります。
3. **セキュリティ：** 「営業部に人事の給与データを見せたくない」といった制限が必要です。

そこで、DWHから必要な分だけを **「小分け」** にして持ってきたのが**データマート**です。

> **例えるなら：**
> - **データウェアハウス（DWH）：** なんでも揃う超巨大な「コストコ」
> - **データマート：** 帰り道にパッと寄れる「コンビニ」や、特定の料理のための「材料セット」

### 整理：データの流れ

一般的なデータの流れは以下のようになります。

1. **データベース（DB）：** 各現場でデータが発生・記録される（レジ、アプリなど）。  
2. **データウェアハウス（DWH）：** 全部のデータをきれいに整えて1ヶ所に集める。
3. **データマート：** 使いやすいように、部署や目的ごとに小分けにする。
4. **分析（BIツール）：** グラフにして意思決定に使う。

--- 
## 4. 関連用語の整理（データレイク・データマート）

データの「加工状態」と「用途」によって呼び方が変わります。

1. **データレイク:** 生のデータをそのまま貯める「池」。
2. **データウェアハウス（DWH）:** 整理整頓された「巨大倉庫」。
3. **データマート:** DWHから「営業部用」「マーケ用」など、特定目的で小分けした「売店」。

---

## 5. ETL：データを運ぶ仕組み

バラバラな場所にあるデータを、DWHで使える形に整えて運び出す工程のことです。

- **E (Extract/抽出):** 各システムからデータを吸い出す。
- **T (Transform/加工):** 形式の統一、お掃除（クレンジング）、計算処理。
- **L (Load/書き出し):** DWHへデータを格納する。
    - _最近のトレンド：先に格納して中で加工する **ELT** も主流。_

## 6. 代表的なETL自動化ツール

手作業やプログラミングを減らし、効率的にデータを連携するためのツールです。

| **カテゴリ**     | **代表的なツール**                    | **特徴**                    |
| ------------ | ------------------------------ | ------------------------- |
| **全自動・海外勢**  | **Fivetran**, **Matillion**    | 設定後の運用が楽、SaaS連携に強い。       |
| **モダン・ELT型** | **dbt**                        | SQLベースで「加工」に特化。品質管理に強い。   |
| **日本国内向け**   | **TROCCO**, **ASTERIA Warp**   | 日本語UI、日本の古いシステムやExcelに強い。 |
| **クラウド純正**   | **AWS Glue**, **BigQuery DTS** | 各クラウドサービスとの親和性が高い。        |
