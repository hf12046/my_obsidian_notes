YAML（ヤムル）とは、**構造化されたデータを人間が読みやすい形式で記述するためのデータフォーマット**のことです。 正式名称は **YAML Ain’t Markup Language** で、あらゆるプログラミング言語に対応しています。 一言で言うと、**「[json](obsidian://open?vault=Obsidian%20Vault&file=json%E3%81%A8%E3%81%AF)の上位互換で、人間にとっての読み書きのしやすさを最優先した記述ルール」** です。

---

##### 1. YAMLの主な特徴

- **可読性が高い:** インデント（字下げ）を使って階層を表現するため、パッと見て構造を理解しやすいです。
- **コメントが可能:** `#` を使ってコメントやメモを残すことができます（JSONにはない機能です）。
- **シンプルな構文:** 閉じ括弧 `}` `]` やタグが不要で、記述量が少なくて済みます。
- **豊富なデータ型:** 文字列や数値だけでなく、日付やNULLなども扱えます。

##### 2. YAMLの書き方（構造）

YAMLは基本的に「キー」と「値」のペアで書かれます。

- **:** **(コロン):** キーと値を区切ります。**コロンの後に必ず半角スペースが必要**です。
- **インデント:** スペース（通常2つ）で階層の深さを表します。**タブ文字は使用できません**。
- **-** **(ハイフン):** リスト（配列）を表します。
- **|** **(パイプ) / >:** 複数行の文章を記述する際に使用します。

###### 具体的な記述例

```
# 顧客情報の設定例
customer:
  name: 山田 太郎
  age: 30
  skills:          # 配列（リスト）
    - Python
    - Docker
  description: |   # 複数行のテキスト
    こちらは備考欄です。
    改行が維持されます。
```

##### 3. 何に使われているの？

YAMLは、人間が手動で編集する「設定ファイル」として、特に以下の分野で標準的に使われています。

1. **インフラ構築・管理 (IaC):**
    - Docker Compose、Kubernetes、Ansibleなどの設定ファイル。
2. **CI/CD (自動化プロセス):**
    - GitHub ActionsやCircleCIなどのワークフロー定義。
3. **アプリケーション設定:**
    - Webフレームワークや静的サイトジェネレータの設定。

##### 4. 他のフォーマットとの違い

YAML、JSON、CSV、XMLは、いずれも**データを保存したり、プログラム間でデータを受け渡したりするための形式**です。

それぞれ「読みやすさ」「記述の自由度」「用途」が異なります。ここでは、**「社員データ」**という同じ内容をそれぞれの形式で記述した例を使って解説します。

---

### 1. JSON (JavaScript Object Notation)

**Web通信のデファクトスタンダード**

現在、Web API（アプリとサーバーの通信など）で最も使われている形式です。JavaScriptの記法がベースですが、多くの言語で扱えます。

- **特徴:** データ量と読みやすさのバランスが良い。
- **構造:** 波括弧 `{}` で囲み、`"キー": 値` のペアで記述します。
- **注意点:** コメント（注釈）が書けません。

JSON

```
[
  {
    "id": 1,
    "name": "山田 太郎",
    "skills": ["Python", "Java"],
    "isActive": true
  },
  {
    "id": 2,
    "name": "佐藤 花子",
    "skills": ["Design", "CSS"],
    "isActive": false
  }
]
```


### 2. YAML (YAML Ain't Markup Language)

**人間にとっての読みやすさを最優先**

設定ファイル（DockerやKubernetes、CI/CDツールなど）でよく使われます。

- **特徴:** 記号が少なく、パッと見て構造がわかりやすい。
- **構造:** インデント（字下げ）で階層を表現します。
- **メリット:** `#` でコメントを書くことができます。

YAML

```
# 社員リスト
- id: 1
  name: 山田 太郎
  skills:
    - Python
    - Java
  isActive: true

- id: 2
  name: 佐藤 花子
  skills:
    - Design
    - CSS
  isActive: false
```

### 3. CSV (Comma-Separated Values)

**表計算ソフトで扱えるシンプルな形式**

ExcelやGoogleスプレッドシートで開けるため、データの集計や移行によく使われます。

- **特徴:** 行と列だけのシンプルな表形式。
- **構造:** カンマ `,` で区切ってデータを並べます。
- **弱点:** 「スキルのリスト」のような**階層構造（ネスト）を持たせるのが苦手**です。（下記例では無理やり文字列として入れています）

コード スニペット

```
id,name,skills,isActive
1,山田 太郎,"Python, Java",true
2,佐藤 花子,"Design, CSS",false
```

### 4. XML (Extensible Markup Language)

**厳密で堅牢な、昔ながらの標準**

HTMLに似たタグ形式です。最近はJSONに押され気味ですが、金融系や大規模システム、Officeファイルの内部構造（.docxなど）で現役です。

- **特徴:** タグで囲むため、どこからどこまでがデータか明確。
- **構造:** `<タグ>値</タグ>` で記述します。
- **弱点:** タグの分だけ文字数が多くなり、ファイルサイズが大きくなりがちです。

XML

```
<employees>
    <employee>
        <id>1</id>
        <name>山田 太郎</name>
        <skills>
            <skill>Python</skill>
            <skill>Java</skill>
        </skills>
        <isActive>true</isActive>
    </employee>
    <employee>
        <id>2</id>
        <name>佐藤 花子</name>
        <skills>
            <skill>Design</skill>
            <skill>CSS</skill>
        </skills>
        <isActive>false</isActive>
    </employee>
</employees>
```

### 比較まとめ

|**特徴**|**JSON**|**YAML**|**CSV**|**XML**|
|---|---|---|---|---|
|**主な用途**|Web API、データ通信|設定ファイル (Config)|データ分析、Excel連携|レガシーシステム、文書|
|**人間からの可読性**|普通|**非常に高い**|普通 (単純な場合)|低い (文字が多い)|
|**機械での処理速度**|**速い**|遅め (解析が複雑)|**非常に速い**|遅め|
|**階層構造**|得意|得意|**苦手**|得意|
|**コメント**|不可|**可**|不可|可|
|**データ量**|普通|少なめ|**最小**|多い|

### 使い分けのヒント

- **プログラムを作って通信させたい** → JSON一択です。
- **ツールの設定ファイルを書きたい** → YAMLが好まれます。
- **データをExcelで管理したい・分析したい** →CSVにします。
- **古いシステムとの連携が必要** → XMLになることがあります。
