## 1. ターミナルとシェルの違い

ひとことで言うと、**「ターミナルは表示画面」**で、**「シェルは命令の実行役」**です。

- **ターミナル（端末 / コンソール）**   
   - 文字を入力するウィンドウ（外枠）のこと。
   - 例：**Windows Terminal**、**Command Prompt（ウィンドウ枠）**、**iTerm2**など。
 
- **シェル（Shell）**
   - ターミナルの中で動き、ユーザーが打ち込んだコマンドを解釈してOSに伝えるプログラム（中身）のこと。   
   - 例：**PowerShell**、**bash (Git Bash)**、**cmd (コマンドプロンプト)**、**zsh**など。
   

### 2. なぜ「ターミナル」と呼ばれてしまうのか？

日常的な会話では、PowerShellを開くことを「ターミナルを立ち上げる」と言っても、エンジニア同士で意味は通じます。
理由は、最近のWindowsでは **「Windows ターミナル」** という一つのアプリの中で、PowerShellやGit Bashを切り替えて使うのが一般的になったからです。

### 比較表

| **名前**            | **種類** | **特徴**                               |
| ----------------- | ------ | ------------------------------------ |
| **PowerShell**    | シェル    | Microsoft製。強力な自動化が可能。                |
| **Git Bash**      | シェル    | Windows上でLinux風の操作（bash）を再現したもの。     |
| **Windows ターミナル** | ターミナル  | PowerShellやGit Bashを「表示する」ための最新のアプリ。 |

- **PowerShell / Git Bash** ＝ 「何ができるか」を決める**中身（シェル）**  
- **ターミナル** ＝ それらを表示する**器（ウィンドウ）**

Windowsで複数のシェル（PowerShell, Git Bash, コマンドプロンプトなど）を使い分けるなら、Microsoft製の「Windows Terminal」をハブ（拠点）にするのが最適解です。


### 画面を分割する方法
Altを押しながら、タブの「＋」ボタンを押す。

### 分割した画面を元に戻す方法
`Ctrl` + `Shift` + `W`（exit入力でもOK）


## シェルの使い分け

正直なところ「どちらでも同じことができる」場面も多いのですが、**「得意料理が違う」**と考えるとスッキリします。
Windowsユーザーにとっての主な3つのシェルの使い分けをまとめました。


### 1. PowerShell（Windowsの守護神）

Windowsを操作・管理するために作られた、非常に強力なシェルです。

- **得意なこと**:
	- Windowsの設定変更（フォルダの一括作成、ユーザー管理など）。
    - Microsoft製品（Excel, Azure, Active Directory）との連携。
    - 複雑なプログラミング（「オブジェクト」という仕組みを使えるため、データの加工に強い）。

- **こんな時に使う**:
    - 「Windows独自のコマンドを動かしたい」
    - 「PC内のファイルを高度に自動処理したい」

### 2. Git Bash（Web開発の標準語）

Linux（リナックス）というOSで使われる「bash」を、Windowsでも使えるようにしたものです。

- **得意なこと**:
    - Gitの操作（`git commit` や `git push` など）。
     - Linuxサーバーと同じコマンド（`ls`, `cd`, `grep`, `ssh` など）での操作。
     - Web開発のチュートリアル記事のコマンドをそのまま写して実行すること。
   
- **こんな時に使う**:
    - 「プログラミングやWebサイト制作をしている時」
    - 「ネット上の解説記事が Linux/Mac 向けに書かれている時」
  

### 3. コマンドプロンプト（古き良き道具）

Windowsの初期からある、最もシンプルなシェルです。

- **得意なこと**:
    - 軽い動作確認（ネットワークがつながっているか確認する `ping` など）。
    - 昔からあるバッチファイル（.bat）の実行。

- **こんな時に使う**:
    - 「古いマニュアルにコマンドプロンプトを使えと書いてある時」
    - 「特に複雑なことをせず、サクッと動作チェックしたい時」


### 結局、どう使い分けるのが正解？

一般的なエンジニアの使い分けスタイルはこんな感じです。

| **目的**                 | **推奨シェル**                        |
| ---------------------- | -------------------------------- |
| **Gitを使ってプログラムを書く**    | **Git Bash**                     |
| **Windowsのシステム設定をいじる** | **PowerShell**                   |
| **どっちを使えばいいか迷った時**     | **PowerShell** (今のWindowsの標準なので) |
### アドバイス：無理に使い分けなくてもOK！

最近のPowerShellは、`ls` や `cd` といったGit Bash（Linux）で使う基本的なコマンドも「エイリアス（別名）」として受け付けてくれるようになっています。
まずは **「基本はPowerShell、Git操作の時だけGit Bash」** と決めて使い始めてみるのが、混乱しなくておすすめです。



## 基本的な操作方法
それぞれのシェルで「これだけは外せない」という基本操作を整理しました。実は、**日常で使う基本的なコマンド（移動や一覧表示）は、PowerShellが気を利かせてGit Bash（Linux）と同じ名前で動くようにしてくれている**ので、そこまで構える必要はありません！


### 1. 共通で使える「超基本」コマンド

まずは、どちらのシェルでも（Windowsのコマンドプロンプトですら）共通して使える必須コマンドです。

|**操作**|**コマンド**|**備考**|
|---|---|---|
|**今いる場所を表示**|`pwd`|Print Working Directoryの略|
|**ファイル一覧を表示**|`ls`|Listの略。中身を確認する|
|**フォルダを移動する**|`cd フォルダ名`|Change Directoryの略|
|**一つ上の階層に戻る**|`cd ..`|`..` は「一つ上」を意味します|
|**画面を綺麗にする**|`clear`|文字が溜まった時にスッキリさせます|
|**終了する**|`exit`|ウィンドウやペインを閉じます|

---

### 2. PowerShell（Windows管理・自動化）

PowerShellは「動詞-名詞」というルールでコマンドが作られているのが特徴です。

- **サービスの状態確認**: `Get-Service`
- **ファイルの中身を読む**: `Get-Content ファイル名`
- **フォルダ作成**: `New-Item -ItemType Directory フォルダ名`
    - ※実は `mkdir フォルダ名` でも動きます（エイリアス機能）。

- **強力なヘルプ**: `Get-Help コマンド名`

> **[Tip]** PowerShellは**「Tabキー」**による予測補完が非常に強力です。フォルダ名の最初の1〜2文字を打って `Tab` を叩くと、自動で補完してくれます。

---

### 3. Git Bash（開発・Linux風操作）

Git BashはLinuxの文化そのものです。

- **空のファイルを作成**: `touch ファイル名.txt`
- **ファイルの中身を画面に出す**: `cat ファイル名`
- **特定の文字を探す**: `grep "検索ワード" ファイル名`
- **管理者権限で実行風（擬似的）**: `sudo` は使えませんが、環境によっては `winpty` を頭につけてWindowsアプリを呼び出すことがあります。
- **Git操作**: `git status` や `git log` など。

> **[Tip]** Git Bash（Linux）では、**大文字と小文字を厳密に区別**します。`Document` と `document` は別物として扱われるので注意してください。

---

### 4. 知っておくとプロっぽい操作術（両方共通）

1. **コマンド履歴の呼び出し**: キーボードの **「↑（上矢印）」** を押すと、さっき打ったコマンドが復活します。
2. **パスの貼り付け**: エクスプローラーのフォルダを、ターミナル画面に**ドラッグ＆ドロップ**してみてください。そのフォルダの「フルパス」が自動入力されます。
3. **強制終了**: 実行中のプログラムが止まらなくなったら **`Ctrl` + `C`**。これが命綱です。
